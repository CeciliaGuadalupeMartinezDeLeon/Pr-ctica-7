FrontEnd

El frontend es la parte del desarrollo web que se dedica a la parte frontal de un sitio web, en pocas palabras del diseño de un sitio web, desde la estructura del sitio hasta los estilos como colores, fondos, tamaños hasta llegar a las animaciones y efectos. Es esa parte de la página con la que interaccionan los usuarios de la misma, es todo el código que se ejecuta en el navegador de un usuario, al que se le denomina una aplicación cliente, es decir, todo lo que el visitante ve y experimenta de forma directa. Un front-end, es la persona que se dedica básicamente al diseño web, pero esto no significa que no toque código, tanto el front-end como el back-end están en contacto con código todo el tiempo.

Dentro del área de front-end se trabaja con lenguajes mayormente del lado del cliente, como:

-HTML (lenguaje de marcado de hipertexto, se utilizan etiquetas que estructuran y organizan el contenido de la web).

-CSS (se encarga del formato y diseño visual de las páginas web escritas en html) para darle estructura y estilo al sitio.

-Javascript (un lenguaje de programación, rápido y seguro para programar centros de datos, consolas, teléfonos móviles o Internet) para complementar los anteriores y darle dinamismo a los sitios web.
-HTML y CSS son lenguajes de marcado y estilo. Javascript es un lenguaje de programación.

¿Para qué sirve el Front end?

El Front end sirve para que un usuario pueda entrar a una web cualquiera, ya que es la parte que este ve y con la que este interactúa al acceder. Debe estar diseñado para ser fácilmente utilizable y también muy atractivo. De hecho, de su usabilidad y su estética depende la imagen que se transmita de la firma en cuestión en el plano digital.



Servidores web

Los servidores web son programas informáticos que almacenan, procesan y entregan páginas web a los usuarios. Los más populares incluyen Apache (un servidor web de código abierto que se utiliza actualmente en el 50% de todos los sitios web) y NGINX, que es bueno para procesos de proxy inverso, almacenamiento en caché, equilibrio de carga y transmisión de medios.



Idiomas y marcos

Una variedad de lenguajes de programación y marcos están involucrados en la construcción del software, también conocido como back-end. Los marcos son bibliotecas de código preescrito con una estructura preestablecida que un desarrollador de back-end puede usar de acuerdo con los requisitos y necesidades. Considerando que, un lenguaje de programación es un superconjunto de lenguajes de scripting como Ruby, Java, Python, PHP, Perl, Erlang y Node.js que se pueden usar para escribir instrucciones para su ejecución.



Sistemas de gestión de bases de datos

Un sistema de gestión de bases de datos (DBMS) es una colección de programas que permite a sus usuarios acceder a una base de datos, manipular, interpretar y representar datos. MySQL es la base de datos relacional de código abierto más popular del mundo. No solo es accesible, sino también gratuito. Su facilidad de configuración y su rendimiento rápido lo convierten en uno de los favoritos entre muchos desarrolladores de backend. Por otro lado, MongoDB es un sistema de base de datos NoSQL de código abierto que está estrechamente asociado con un conjunto de tecnologías basadas en JavaScript como ExpressJS, AngularJS y NodeJS.



Servicios de colaboración

Dado que las operaciones de back-end y front-end van de la mano, es ideal que los desarrolladores de ambos lados permanezcan conectados entre sí. Plataformas como Slack, Asana, Jira y Trello ayudan a los equipos a trabajar de forma eficaz con una mejor colaboración.



Entornos de desarrollo local

Todos los desarrolladores de back-end jurarán por la importancia de un entorno de prueba local. La ventaja de utilizar un sitio local que solo usted puede ver le da la libertad de probar códigos y experimentos antes de que el sitio entre en funcionamiento. XAMPP y WampServer son ejemplos de entornos de desarrollo de Windows de código abierto que permiten a los usuarios utilizar aplicaciones web con bases de datos Apache, PHP y MySQL.



Verificadores de rendimiento del sitio web

En la era digital, la capacidad de atención promedio de un ser humano es de apenas 8 segundos. Por lo tanto, la velocidad a la que se carga un sitio influye en gran medida en el tráfico del sitio. Incluso un retraso de un segundo puede hacer que un cliente se despida de su sitio. Es por eso que los desarrolladores de back-end consideran a los probadores de velocidad como Google PageSpeed ​​Insights y Full-Page Load Tester como importantes herramientas de desarrollo de back-end. Estos no solo brindan informes de la velocidad del sitio, sino que también recomiendan ajustes que puede hacer para que su página sea más rápida.
